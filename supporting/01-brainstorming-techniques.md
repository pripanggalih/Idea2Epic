# Brainstorming Techniques Reference

A collection of proven brainstorming techniques to use with the Brainstorming Agent.

---

## 1. SCAMPER Method

Use these prompts to explore ideas systematically:

| Letter | Prompt | Example Questions |
|--------|--------|-------------------|
| **S** - Substitute | What can be replaced? | What materials, people, processes can be substituted? |
| **C** - Combine | What can be merged? | What ideas, features, or purposes can be combined? |
| **A** - Adapt | What can be adjusted? | What else is like this? What could be copied or adapted? |
| **M** - Modify | What can be changed? | What can be magnified, minimized, or altered? |
| **P** - Put to other uses | What else can it do? | What other markets or uses could this serve? |
| **E** - Eliminate | What can be removed? | What's unnecessary? What if we simplified? |
| **R** - Reverse/Rearrange | What if we flip it? | What if we did the opposite? Changed the order? |

## 3. "What If" Scenarios

Challenge assumptions with these prompts:

- What if we had unlimited budget?
- What if we had to launch in 1 week?
- What if our biggest competitor copied this tomorrow?
- What if our target user was 10 years older/younger?
- What if we removed the most "essential" feature?
- What if we had to charge 10x the price?
- What if this had to work offline?
- What if we could only use voice/no screen?

---

## 4. Reverse Brainstorming

Instead of "How do we solve X?", ask:
1. "How could we make X worse?"
2. "How could we cause X to fail?"
3. Then reverse each answer into a solution.

**Example:**
- Problem: "How to improve customer satisfaction?"
- Reverse: "How to make customers hate us?"
  - Make them wait forever ‚Üí Solution: Reduce wait times
  - Ignore their complaints ‚Üí Solution: Responsive support
  - Make product confusing ‚Üí Solution: Simplify UX

## 5. Mind Mapping Structure

Start from center, branch out:

```
                    [Feature A]
                   /
    [User Need 1]--[Feature B]
   /               \
[CORE IDEA]         [Feature C]
   \
    [User Need 2]--[Feature D]
                   \
                    [Feature E]
```

**Rules:**
- One word/short phrase per branch
- Use colors for different themes
- Connect related ideas across branches
- No judgment during creation

---

## 6. 5 Whys Technique

Dig deeper into problems:

1. **Why** does this problem exist? ‚Üí [Answer 1]
2. **Why** is [Answer 1] happening? ‚Üí [Answer 2]
3. **Why** is [Answer 2] happening? ‚Üí [Answer 3]
4. **Why** is [Answer 3] happening? ‚Üí [Answer 4]
5. **Why** is [Answer 4] happening? ‚Üí [Root Cause]

---

## 7. Random Word Association

1. Pick a random word (noun works best)
2. List attributes of that word
3. Force connections to your problem

**Example:**
- Random word: "Restaurant"
- Attributes: Menu, reservations, chef, courses, tips
- Problem: "Improve onboarding"
- Connections:
  - Menu ‚Üí Offer choices in onboarding path
  - Reservations ‚Üí Schedule personalized sessions
  - Courses ‚Üí Break into digestible steps
  - Tips ‚Üí Add helpful hints throughout

## Quick Reference: Idea Capture Format

For each idea generated, capture:

```
**[Category] Idea Name**
- Concept: [2-3 sentence description]
- Value: [Why this matters to users]
- Novelty: [What makes this different]
```

---

## Session Tips

1. **Quantity over quality** - Aim for 50+ ideas before filtering
2. **No judgment** - Write everything down, even "bad" ideas
3. **Build on others** - "Yes, and..." not "No, but..."
4. **Take breaks** - Fresh eyes find new angles
5. **Mix techniques** - Different methods unlock different thinking
