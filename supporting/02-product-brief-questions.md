# Product Brief Discovery Questions

A reference guide for the Product Brief Agent to conduct thorough product discovery.

---

## Vision Discovery Questions

### Problem Understanding
- What core problem are you trying to solve?
- Who experiences this problem most acutely?
- How do people currently solve this problem?
- What's frustrating about current solutions?
- What happens if this problem goes unsolved?

### Solution Vision
- If we could solve this perfectly, what would that look like?
- What's the simplest way we could make a meaningful difference?
- What would make users say "this is exactly what I needed"?
- What excites you most about this solution?

### Competitive Landscape
- What solutions exist today?
- Where do they fall short?
- What gaps are they leaving open?
- Why haven't existing solutions solved this completely?

### Unique Differentiators
- What's your unfair advantage?
- What would be hard for competitors to copy?
- What insight or approach is uniquely yours?
- Why is now the right time for this solution?

---

## Target User Questions

### User Identification
- Who is your primary user? Describe them.
- What's their role/job/situation?
- What are they trying to accomplish?
- What frustrates them most about current options?

### User Context
- Where/when do they experience this problem?
- How often do they encounter this problem?
- What's the cost (time/money/stress) of the current situation?
- What have they tried before?

### User Motivation
- What would success look like for them?
- What would make them switch from current solution?
- What would make them recommend this to others?
- What would make them pay for this?

## Success Metrics Framework

### User-Centric Metrics
| Category | Example Metrics |
|----------|-----------------|
| Adoption | Sign-ups, activations, daily/monthly active users |
| Engagement | Session duration, feature usage, return visits |
| Satisfaction | NPS, CSAT, reviews, referrals |
| Retention | Churn rate, renewal rate, lifetime value |

### Business Metrics
| Category | Example Metrics |
|----------|-----------------|
| Revenue | MRR, ARR, average deal size |
| Growth | User growth rate, viral coefficient |
| Efficiency | CAC, LTV/CAC ratio, payback period |
| Market | Market share, competitive win rate |

### Product Metrics
| Category | Example Metrics |
|----------|-----------------|
| Performance | Load time, uptime, error rate |
| Quality | Bug count, support tickets, crash rate |
| Velocity | Time to value, onboarding completion |

## Scope Definition Framework

### In Scope Checklist
Ask for each proposed feature:
- [ ] Does this directly solve the core problem?
- [ ] Is this essential for the primary user?
- [ ] Can we launch without this? If yes, consider moving to "out of scope"
- [ ] Do we have clarity on what this means?

### Out of Scope Categories
Common items to explicitly exclude:
- Future phases / "nice to haves"
- Edge case handling (for v1)
- Platform variations (e.g., "mobile app later")
- Integrations (unless core to value)
- Admin/back-office features (unless essential)
- Advanced customization options

## Risk Assessment Framework

### Risk Categories

| Category | Questions to Ask |
|----------|------------------|
| **Market** | Is there demand? Will people pay? |
| **Technical** | Can we build this? Do we have the skills? |
| **Resource** | Do we have time/budget/people? |
| **Competitive** | Can competitors respond quickly? |
| **Regulatory** | Are there legal/compliance concerns? |
| **Adoption** | Will users actually change behavior? |

## Product Brief Quality Checklist

Before finalizing, verify:

### Clarity
- [ ] Can someone unfamiliar understand the product in 2 minutes?
- [ ] Is the problem statement specific and measurable?
- [ ] Is the target user clearly defined?

### Completeness
- [ ] Problem, solution, and value proposition are defined
- [ ] Success metrics are specific and measurable
- [ ] Scope boundaries are clear (in/out)
- [ ] Key assumptions are documented
- [ ] Major risks are identified with mitigations

### Alignment
- [ ] Solution directly addresses the stated problem
- [ ] Features map to user needs
- [ ] Metrics measure what matters
- [ ] Scope is achievable with available resources
